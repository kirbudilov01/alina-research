# ALINA_RESEARCH_OS

Market research operating system for the Alina concept app.

## Objective

Build evidence-first product strategy for an app at the intersection of:

1. Gaming
2. Astrology/Esoterics
3. Avatar Identity
4. Coaching
5. Mindfulness

Core user loop hypothesis: date of birth based interpretation + daily avatar reinforcement of the user's "best self".

## Research Flow

1. Formulate hypothesis v1.
2. Scan top apps by category (Top 200 each).
3. Correct hypothesis from evidence.
4. Define target markets and validate market existence.
5. Run global competitive analysis.
6. Find whitespace and formulate hypothesis vN.
7. Produce go/no-go decision pack.

## Repository Structure

- `docs/` research docs and synthesis
- `sources/` source registry and citation protocol
- `templates/` reusable templates
- `data_raw/` raw extracted data
- `data_processed/` normalized analysis tables
- `reports/` final reports and executive packs
- `hypotheses/` versioned hypothesis snapshots

## Source Rule

Every important claim must include source id(s), confidence, and last update date.

Format:

`Claim text (sources: SRC-XXXX, confidence: High|Medium|Low, updated: YYYY-MM-DD)`

## Delivery Cadence

- Daily: progress summary + blockers + next steps.
- Milestone: hypothesis revision after each major evidence phase.
- Final: go/no-go memo and follow-up roadmap.
